Lesson 1 — The Canvas and Coordinates

The Canvas
An HTML <canvas> element is a rectangle of pixels we can control from JavaScript. We ask for a "2D rendering context" and get back an object with drawing commands: fillRect(), arc(), stroke(), etc. The browser handles turning those commands into pixels on screen.
The Coordinate System
Every pixel has an address (x, y). The origin (0, 0) is the top-left corner. x increases going right; y increases going down. This is the opposite of math-class coordinates where y goes up — keep that in mind for Lesson 3 when we enter 3D. Move your mouse over the canvas to see live coordinates.
The Game Loop
A game repeats three steps ~60 times per second:
  1. Update — advance the world (move objects, check rules)
  2. Draw — render everything to the canvas
  3. Yield — hand control back; the browser calls us for the next frame
requestAnimationFrame(callback) schedules our callback for the next frame. It receives a timestamp in milliseconds.
Delta Time (dt)
dt = elapsed seconds since the last frame (~0.016 at 60 fps). We multiply every movement by dt so the game runs at the same speed on fast and slow machines. "Move 80 pixels per second" = move 80 × dt pixels this frame.
